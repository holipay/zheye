## 1. 关键字关联系统

### 背景

文章入库后需要长期保存，并建立文章间的关联关系，便于后续检索和分析。

### 实现方案

采用 **关键字 + 关联表** 混合方案：

- 专业术语词库（268 个经济/金融/社科术语）
- 基于关键字重叠度计算文章关联（Jaccard 系数）
- 阈值：score > 0.3 生成关联

### 新增文件

| 文件 | 说明 |
|------|------|
| `configs/keywords.yaml` | 专业术语词库 |
| `models/keyword.py` | Keyword 模型 |
| `models/article_keyword.py` | ArticleKeyword 关联模型 |
| `models/article_relation.py` | ArticleRelation 关联模型 |
| `scraper/pipeline/keywords.py` | 关键字匹配模块 |
| `scraper/pipeline/relations.py` | 关联计算模块 |
| `migrations/versions/002_add_keywords.sql` | 数据库迁移脚本 |
| `scripts/backfill_keywords.py` | 历史数据回填脚本 |

### 关键字匹配规则

- 英文：词边界正则匹配（`\bterm\b`）
- 中文：精确词组匹配
- 分类过滤：优先匹配同分类关键字
- 长度过滤：中文 ≥ 2 字，英文 ≥ 3 字符

### API 接口

| 接口 | 说明 |
|------|------|
| `GET /api/news/{id}/related` | 获取关联文章 |
| `GET /api/keywords` | 关键字列表（支持分类/语言过滤） |
| `GET /api/keywords/popular` | 热门关键字（按文章数排序） |
| `GET /api/keywords/{id}/articles` | 按关键字查文章 |

## 2. 移除翻译功能

### 原因

翻译功能存在以下问题：
- 增加抓取延迟
- 依赖外部 API（MyMemory）
- 翻译质量不稳定
- 关键字词库已覆盖中英文，匹配不受影响

### 修改内容

| 文件 | 修改 |
|------|------|
| `scraper/run_news.py` | 移除翻译调用 |
| `scraper/pipeline/__init__.py` | 移除 translate 导出 |

### 效果

- 抓取速度提升
- 无翻译失败导致的错误
- 新文章的 `translated_title` 为 null
- 历史翻译数据保留不变

## 3. 双排序模式

### 功能

支持两种排序方式：

| 排序 | 参数 | 说明 |
|------|------|------|
| 按日期 | `?sort=date` | 最新优先（默认） |
| 按权重 | `?sort=weight` | 高权重优先 |

### 权重计算

文章权重 = 匹配关键字的权重之和

示例：
- 文章匹配 GDP(2.0) + 通胀(1.8) + 美联储(2.0) = 5.8

### API 变更

`/api/news` 接口新增参数：
- `sort`: 排序方式（date/weight）
- `keyword_id`: 按关键字筛选

## 5. 反爬措施

### 问题

固定时间抓取可能触发网站反爬机制。

### 改进方案

| 措施 | 实现 | 说明 |
|------|------|------|
| User-Agent 轮换 | ✅ | 8 个 UA 随机选择 |
| 请求随机延迟 | ✅ | 1-4 秒随机延迟 |
| 域名限流 | ✅ | 同域名 5-12 秒间隔 |
| 并发控制 | ✅ | 最多 5 个并发请求 |
| 分批处理 | ✅ | 每批 5 个源，批间 10-30 秒延迟 |
| 源顺序随机 | ✅ | 每次打乱源顺序 |

### 抓取流程

```
24 个源 → 打乱顺序 → 分 5 批
                        ↓
        批次1 (5个源) → 等待 10-30s
        批次2 (5个源) → 等待 10-30s
        ...
        批次5 (4个源)
                        ↓
                    保存到数据库
```

### 文件变更

| 文件 | 修改 |
|------|------|
| `scraper/sources/fetcher.py` | UA 轮换、延迟、并发控制 |
| `scraper/run_news.py` | 分批处理、随机顺序 |
